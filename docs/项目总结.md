# 考公信息运营 Agent 系统 — 项目总结

**版本**: v2.0 ｜ **日期**: 2026-06-17 ｜ **代码量**: 15模块 / 8000+行

---

## 一句话描述

> 从零构建了一套 **Multi-Agent 智能协作系统**，覆盖数据采集→RAG 语义搜索→知识图谱→内容生成→质量审核→自动发布的全链路，支持 Web/微信双端 AI 对话。

---

## 核心能力

### 1. RAG 语义搜索引擎

```text
BAAI/bge-small-zh-v1.5 (512维) × FAISS 向量索引 × RRF 混合检索

"海淀区有没有计算机类的事业编"
  → 语义理解 "海淀区"∈"北京"、"计算机类"→正文匹配、"事业编"→"事业单位"
  → 向量相似度 top-5 + SQL 精确 top-5 → RRF 融合 → 综合排序
  → 检索 <50ms，737 条公告已索引
```

### 2. 知识图谱引擎

```text
SQLite 轻量图数据库：1524 实体节点 + 2211 关系边

Region(36) → BELONGS_TO → Announcement(737) ← PUBLISHES ← Organization(737)
                               │
                          HAS_CATEGORY
                               │
                          ExamType(14)

支持：实体搜索 / 关联推荐 / 同地区同类型发现
```

### 3. Multi-Agent 协作

```text
MultiAgentOrchestrator
  ├── CrawlerAgent    → 采集+搜索原文
  ├── EditorAgent     → 标题模板+适合人群提醒+正文
  ├── QAAgent         → DeepSeek质检+事实评测+诊断
  └── PublisherAgent  → 微信草稿箱+Token管理

通信协议: AgentMessage { msg_id, role, task, payload, reasoning, status, elapsed }
```

### 4. 三层 AI 对话路由

```text
用户消息
  ├── L1: 关键词匹配 (<10ms)       → 地区+类型识别 → SQL精确查 → 立即返回
  ├── L2: RAG+KG+DeepSeek (1-5s)   → 语义检索公告原文 → 知识图谱推荐 → AI生成
  └── L3: 兜底回复 (<1s)           → 引导用户正确使用
```

## 技术栈

```text
语言/框架: Python 3.8+, http.server (标准库)
向量数据库: FAISS (Facebook), bge-small-zh-v1.5 (512维)
图数据库: SQLite + NetworkX (轻量图)
大模型: DeepSeek API (v4-flash/v4-pro)
工作流: LangGraph (状态图)
浏览器: Playwright (Chromium)
流式推送: SSE (Server-Sent Events)
数据存储: SQLite (WAL模式, 15张表)
```
