## 一、RAG（检索增强生成）可衍生功能

### 当前瓶颈

现在的考公 Agent 回复用户时，要么靠关键词匹配查数据库，要么纯靠 DeepSeek 的通用知识，存在明显短板：

| 问题 | 现状 | RAG 改善 |
|------|------|----------|
| 地问法多样 | "北京事业编"可以，但"海淀区有没有招计算机的体制内岗位"匹配不了 | 向量语义检索 |
| 通用知识不准 | DeepSeek 不知道"2026年国考报名截止时间" | 检索实时公告后回答 |
| 政策解读 | 无法回答"应届生怎么认定""基层工作经历怎么算" | 检索公告正文 + 官方文件 |
| 备考指导 | 不能给针对性建议 | 检索历史优质内容 |

### 功能 1: 公告语义搜索引擎

**原理**: 把所有公告正文做向量化（Embedding），用户自然语言 → 语义相似度检索 → 返回最匹配的公告。

```
用户: "海淀区有没有招计算机的体制内岗位"

传统: 关键词匹配 → 失败（"体制内"≠"事业单位"，"计算机"可能不在标题中）
         ↓
RAG:  "海淀区有没有招计算机的体制内岗位"
         ↓ Embedding (text-embedding-3-small 或 bge-large-zh)
         ↓ 向量相似度 top-5
         ↓ 找到: 海淀区2026年事业单位招聘...（正文含"计算机类"）
         ↓ DeepSeek 基于检索结果生成自然回复
```

**新增模块**: `kaoyan_collector/rag_search.py`

```python
class AnnouncementRAG:
    """公告语义检索引擎"""

    def __init__(self):
        self.embeddings = self._load_or_build_index()

    def build_index(self):
        """对所有公告正文做向量化，存入 ChromaDB / FAISS"""
        events = fetch_all_events_with_text()
        texts = [f"{e.title} {e.region} {e.category} {e.raw_text[:500]}" for e in events]
        vectors = embedding_model.encode(texts)
        self.index = FAISS.index_factory(vectors)

    def search(self, query: str, top_k: int = 5) -> list[SearchResult]:
        """语义搜索公告"""
        query_vec = embedding_model.encode(query)
        scores, indices = self.index.search(query_vec, top_k)
        return [self.events[i] for i in indices[0]]
```

**面试价值**: "我引入了 RAG 技术，用户可以用自然语言搜索 737 条公告，不需要知道确切的地区名或考试类型名"

### 功能 2: 智能问答（基于公告全文）

**原理**: 检索到相关公告后，把公告原文作为 context 注入 DeepSeek，让 AI 基于原文回答。

```
用户: "河北省教师招聘要求本科学历吗"

RAG流程:
  Step 1: 向量检索 → 找到 5 条河北教师招聘公告原文
  Step 2: 拼接 prompt:
      "请基于以下公告原文回答用户问题：{问题}
       相关公告：{检索到的公告原文}
       如果公告中没有明确说明，请如实告知"
  Step 3: DeepSeek 生成: "根据近期河北教师招聘公告，学历要求因岗位而异：
       中小学教师岗位一般要求本科及以上，部分乡镇学校可放宽至大专。
       具体请以各公告原文为准。"

聊天记录保留引用来源：
  🤖: 根据以下公告回答...
      📎 来源1: 石家庄市2026年教师招聘公告
      📎 来源2: 河北省直属事业单位招聘公告
```

**新增功能**: 在聊天回复中显示引用来源，点击可跳转到公告详情页

### 功能 3: 个性化备考推荐

**原理**: 用户告诉 Agent 自己的背景（学历、专业、地区偏好），Agent 用 RAG 匹配最合适的正在报名公告。

```
用户: "我是计算机硕士，想在广东找体制内工作"

RAG流程:
  Step 1: 从用户画像提取关键信息（学历=硕士, 专业=计算机, 地区=广东）
  Step 2: 在公告库中向量检索 + 结构化过滤
  Step 3: 输出匹配结果:

  🤖: 根据你的背景，为你匹配到以下机会：
      1. ⭐⭐⭐ 广东省2026年选调生（计算机类，硕士，正在报名）
      2. ⭐⭐ 深圳市事业单位招聘（计算机岗，3天后截止）
      3. ⭐ 广州某国企信息技术岗（本科及以上，报名中）

      其中选调生和你的背景最匹配，建议优先准备。
```

**用户画像存储**: 在 `agent_memory.py` 中专存一个 `user_profile` 记忆类型

### 功能 4: 政策文件 RAG 问答

**原理**: 收集考公相关的政策文件（公务员法、事业单位管理条例、各省招考公告中的政策说明），构建独立的政策知识库 RAG。

```
用户: "什么是基层工作经历？我怎么证明我有？"

政策RAG: 检索政策库 → 找到对应条款 → DeepSeek总结 + 引用原文

🤖: 根据《公务员录用规定》，基层工作经历是指：
    在县级及以下党政机关、国有企事业单位、村（社区）组织、
    其他经济组织和社会组织等工作的经历。
    📎 来源: 《公务员录用规定》第十六条
    你也可以参考各省公告中的具体认定标准。
```

**数据来源**: 
- 国家公务员局政策法规栏
- 各省人事考试网的招考公告中的"报考条件"章节
- 人力资源和社会保障部相关规定

## 二、知识图谱可衍生功能

### 当前数据关系（未建模）

现在公告数据是扁平表，但考公信息之间存在丰富的关系：

```
省级人社厅 ──发布──→ 省考公告
    │
    └──下属──→ 市级人社局 ──发布──→ 市事业单位公告
                    │
                    └──下属──→ 区县人社局 ──发布──→ 教师招聘
                    
考试类型层级:
  公务员 ──├── 国考
         ├── 省考
         └── 选调生

  事业单位 ──├── 教师
           ├── 医疗
           └── 其他

地区层级:
  中国 ──├── 华北 ──├── 北京
        │         ├── 天津
        │         └── 河北 ──├── 石家庄
        │                   └── 保定
        ├── 华东 ...
        └── ...
```

### 功能 5: 考公职位关系图谱

**原理**: 用 Neo4j 或 NetworkX 构建实体-关系图，节点间有关联。

```
图谱节点:
  - 地区 (Region)
  - 单位 (Organization)  
  - 公告 (Announcement)
  - 考试类型 (ExamType)
  - 岗位类别 (PositionCategory)

图谱关系:
  - Organization --BELONGS_TO--> Region
  - Organization --PUBLISHES--> Announcement  
  - Announcement --BELONGS_TO--> ExamType
  - Announcement --HAS_DEADLINE--> Date
  - Announcement --SIMILAR_TO--> Announcement (基于内容相似度)
```

**可视化举例**:

```
         ┌─────────────┐
         │  河北省人社厅  │
         └──────┬──────┘
                │ 发布
    ┌───────────┼───────────┐
    ▼           ▼           ▼
 [省考公告] [事业编公告] [教师招聘]
    │           │           │
    ├── 招120人  ├── 招52人   └── 招15人
    ├── 3天后截止 ├── 今天截止
    └── 计算机类  └── 本科及以上
```

### 功能 6: "相关公告"智能推荐

**原理**: 用户查看某条公告时，基于知识图谱推荐相关公告。

```
推荐逻辑:
  同一单位发布的其他公告
  → 同一地区同类型公告
  → 招聘人数相似的公告
  → 截止日期接近的公告
  → 岗位类别交叉的公告
```

```
┌─────────────────────────────────────────────┐
│  石家庄市2026年事业单位招聘公告              │
│  招52人 | 6月20日截止 | 本科及以上           │

│  📌 相关公告：                               │
│  ● 河北省直属事业单位招聘（同省，正在报名）   │

  Step 2: 按时间轴排列
  Step 3: 按单位类型、考试类型分组
  
  🤖: 根据历史数据（2025-2026），计算机专业在河北的招聘呈以下趋势：
      ● 2026年H1 共有 37 条相关公告，同比增长 15%
      ● 主要集中在：事业单位(18条) > 国企(12条) > 公务员(7条)
      ● 招聘高峰期：3月(省考)、6月(事业单位)
      ● 平均招聘人数：28人/公告
      ● 热门岗位：信息技术岗、数据分析岗、网络安全岗
```

## 三、技术选型建议

| 技术 | 用途 | 推荐方案 | 替代方案 |
|------|------|----------|----------|
| 向量数据库 | RAG 检索 | **ChromaDB** (lightweight, Python native, SQLite 后端) | FAISS, Qdrant |
| Embedding 模型 | 文本向量化 | **bge-large-zh-v1.5** (中文最优, 免费) | text-embedding-3-small (付费) |
| 图数据库 | 知识图谱 | **NetworkX + SQLite** (零额外依赖) | Neo4j (重) |
| 向量检索 | 语义搜索 | **ChromaDB 内置** | FAISS |
| 文档解析 | 政策文件 | **现有 Attachment 解析能力** | unstructured, langchain |

## 六、与当前对话方案的整合

```
当前设计方案                    加上 RAG + KG 后
─────────────                  ────────────────
意图识别 (关键词)             意图识别 (关键词 + 向量语义)
     ↓                              ↓
数据库查询 (SQL)              混合检索 (SQL + 向量 + 图谱)
     ↓                              ↓
直接返回结果                   知识图谱关联推荐
     ↓                              ↓
DeepSeek 生成回复              RAG 注入上下文 + 来源引用
```

即在现有三层路由之上增加 **语义层**，让"查不到"的情况大幅减少。

---

**总结**: RAG 和知识图谱可以让项目从"能查到就查到"升级为"总能找到相关答案"，大幅提升面试时的技术深度展示。
