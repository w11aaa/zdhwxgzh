## 一、项目概述

### 1.1 项目名称

考公信息运营 Agent 系统（Gongkao Content Operation Agent System）

### 1.2 项目背景

公务员、事业单位、教师、国企等招聘公告分布在 20 余个网站（国家公务员局、各省人事考试网、事业单位招聘平台、高校就业网等），存在以下核心问题：

- **信息高度分散**：同一考试类型的公告分布在不同网站，格式不统一，用户获取成本极高
- **时效极度敏感**：公告报名有严格截止日期，错过截止即失去报考资格
- **附件处理复杂**：岗位表散落在 .xlsx / .docx / .pdf 中，人工下载解析效率低
- **内容合规要求高**：招聘公告属于事实敏感内容，AI 生成时不能凭空补充或改写
- **人工重复劳动严重**：微信公众号运营人员每天需要花费 2-3 小时在查找、整理、排版的重复性工作上

### 1.3 建设目标

构建一套完整的 **Multi-Agent 智能协作系统**，实现从数据采集、结构化入库、智能检索、内容生成、质量审核到自动发布的全链路自动化，将传统需要 3 小时的人工操作压缩至 5 分钟内完成。

### 1.4 项目范围

- **数据源**：粉笔考试信息 API（主力）、公考雷达网站（备用）
- **考试类型**：公务员（国考/省考）、事业单位、国企、教师、医疗、选调生等 18 种
- **发布渠道**：微信公众号（草稿箱模式）、Web 对话面板
- **用户角色**：内容运营人员（管理员）

## 三、功能需求

### 3.1 数据采集模块

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-01 | 粉笔 API 采集 | 调用粉笔考试信息 API，按考试类型、报名状态、页码采集公告并入库 | P0 |
| F-02 | 公考雷达采集 | 通过 Playwright 浏览器自动化从公考雷达网站采集公告（备用） | P1 |
| F-03 | 一键补全 | 遍历全部 18 种考试类型，自动采集全国范围内"即将开始"和"正在报名"的公告 | P0 |
| F-04 | Hash 去重 | 基于 source_platform + source_id 组合主键去重，避免重复入库 | P0 |
| F-05 | 状态自动修正 | 根据报名截止日期与当前日期比较，自动修正报名状态（即将开始/正在报名/报名结束） | P0 |
| F-06 | 原文搜索 | 通过标题在 DuckDuckGo 搜索官方原文链接，使用 Playwright 抓取原文 HTML | P1 |

### 3.2 智能检索模块（RAG）

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-07 | 向量索引构建 | 使用 bge-small-zh-v1.5 对所有公告正文做 Embedding，构建 FAISS 向量索引 | P0 |
| F-08 | 语义搜索 | 支持自然语言查询，如"不限专业的招聘""快截止的教师岗"，基于向量语义相似度检索 | P0 |
| F-09 | 混合检索 | 向量语义路 + 关键词精确路双路召回，通过 RRF 算法融合排序 | P0 |
| F-10 | 索引持久化 | 向量索引缓存到磁盘，重启后自动加载（4.6s）而无需重建（22s） | P1 |

### 3.3 知识图谱模块

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F-11 | 实体自动抽取 | 从公告数据中自动提取地区、单位、考试类型等实体节点 | P0 |
| F-12 | 关系自动构建 | 自动构建 BELONGS_TO / PUBLISHES / HAS_CATEGORY / SIMILAR_TO 关系边 | P0 |
| F-13 | 关联推荐 | 用户查看某条公告时，基于图关系推荐同地区同类型的其他公告 | P1 |

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 | 当前达成 |
|------|------|----------|
| RAG 索引加载 | < 10 秒 | 4.6 秒 |
| RAG 单次检索 | < 100ms | < 50ms |
| 关键词查询 | < 100ms | < 10ms |
| 对话 DeepSeek 回复 | < 8 秒 | 1-5 秒 |
| Agent Planner 全程 | < 120 秒/条 | 30-60 秒 |
| Web 页面响应 | < 1 秒 | < 100ms |

### 4.2 可靠性要求

- 单条公告采集失败不影响整体采集流程
- DeepSeek API 调用超时自动回退到关键词匹配
- 数据库 WAL 模式，支持并发读写不锁表
- Agent 任务失败自动记录诊断信息

### 4.3 安全性要求

- DeepSeek API Key 存储在本地文件，不传入代码仓库
- 微信公众号 AppSecret 存储在本地配置文件
- 质检不通过自动阻断发布，防止内容合规风险

### 4.4 可维护性要求

- 单文件 HTTP Server，无复杂部署依赖
- 数据库自动迁移（DDL + ALTER TABLE），升级无感知
- 清晰的模块边界（Agent/Engine/Service/UI 分层）

### 4.5 兼容性要求

- 操作系统：Windows 10+ / Linux / macOS
- Python 版本：3.8+
- 数据库：SQLite 3.x（WAL 模式）
