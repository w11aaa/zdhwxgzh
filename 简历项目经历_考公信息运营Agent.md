# 考公信息运营 Agent

**个人项目** ｜ 2025.06 – 至今

**技术栈：** Python、SQLite、DeepSeek API、微信公众号 API、Playwright、HTTP Server

**项目地址：** https://github.com/w11aaa/zdhwxgzh

---

## 项目概述

面向微信公众号内容运营场景的智能 Agent 系统。自动从粉笔、公考雷达等多源采集公务员/事业单位/教师/国企等招聘公告（已入库 737 条），经 Agent Planner 动态编排工具链路，自动生成公众号文章并提交草稿箱，全流程带可审计决策记录。

---

## 核心职责与技术实现

**1. 多源数据采集与结构化**

设计粉笔 API 和公考雷达双源采集策略，支持 18 种考试类型分页采集、一键补全全国有效公告。基于 Hash 去重和截止日期自动修正报名状态（即将开始/正在报名/报名结束），通过 DuckDuckGo + Playwright 自动搜索官方原文链接，原文覆盖率 96.5%（711/737）。

**2. 动态 Agent 工具编排器**

自研 Agent Planner：根据每条公告的实时状态（原公告是否已找到、附件是否已扫描/下载/解析），动态生成工具调用计划。例如对未找到原文的公告自动触发搜索，对有待下载岗位表的自动触发附件下载，每步记录决策原因（reasoning）和执行耗时到 agent_steps 审计表，实现完全可追溯的 Agent 决策链路。

**3. 大模型内容生成与质量保障**

基于公告结构化字段 + 原公告 HTML 自动生成公众号文章，内置标题模板引擎（地区+人数+岗位亮点+截止时间）和"适合人群提醒"模块（根据学历/专业/考试类型/招聘规模自动匹配）。接入 DeepSeek 模型进行草稿质检，检查无关内容、事实篡改、原文链接遗漏，质检不通过自动阻断发布。配套事实一致性评测脚本，抽样评测招聘人数准确率、截止日期准确率、原文链接保留率等 5 个维度。

**4. 失败自动诊断与恢复**

自建诊断引擎覆盖微信公众号 API 全部错误码（access_token 过期/权限不足/IP 白名单等）和 8 类通用异常模式（网络连接/DNS/超时/数据库锁/Playwright 崩溃等），每种错误匹配可能原因、修复建议和自动恢复策略（如自动重新获取 Token、跳过非真实附件链接）。

**5. 附件智能处理**

从公告原文 HTML 中自动提取附件链接，按文件名规则筛选岗位表（排除报名表/承诺书/专业目录等无关附件），支持 .xlsx/.docx/.pdf 下载和解析，将岗位表渲染为公众号可直接插入的图片。已登记 1855 条附件记录。

**6. 可视化控制台与微信公众号集成**

自建 Web Dashboard（纯 Python HTTP Server），支持采集任务编排、公告库检索筛选、批量生成预览/提交草稿、Agent 运行记录查看、工具注册表展示。通过微信公众号草稿接口完成真实发布链路（已提交 13 条草稿），采用"自动提交草稿箱 + 人工确认发布"的人机协同安全模式。

## 项目亮点（面试建议话术）

> 1. **不只是爬虫**：这是一个完整的 Agent 系统——感知环境（采集+状态判断）、规划执行（Planner 动态编排）、行动输出（生成+发布）、自我反思（质检+诊断）。
>
> 2. **可审计的 Agent 决策**：每一步工具调用都记录了 reasoning（为什么调这个工具）、参数、结果和耗时，存入 agent_runs/agent_steps 表，面试时可以直接展示数据库里的决策链路。
>
> 3. **工程可靠性设计**：专门的评测脚本量化内容质量，诊断模块让系统从"报错"升级为"能解释为什么出错并给出修复建议"。
>
> 4. **真实业务闭环**：不是 Demo，接入了真实的微信公众号 API，系统每天可以自动产出可直接发布的草稿。
