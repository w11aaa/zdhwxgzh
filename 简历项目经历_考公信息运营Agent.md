# 考公信息运营 Agent 系统

**个人项目** ｜ 后端负责人 & 架构设计 ｜ 2025.06 – 至今

**技术栈：** Python、SQLite、FAISS、DeepSeek API、LangGraph、BGE Embedding、NetworkX、Playwright、SSE、微信公众号 API、HTML/CSS/JS

**项目地址：** https://github.com/w11aaa/zdhwxgzh

---

## 项目描述

面向微信公众号内容运营场景的智能 Agent 系统。从粉笔、公考雷达等多源采集公务员/事业单位/教师/国企等招聘公告（已入库 737 条），基于 RAG 语义搜索 + 知识图谱构建自然语言查询能力，通过 Multi-Agent 协作自动完成内容生成、质检、发布全流程。支持 Web/微信双端对话。

---

## 主要工作

**1. Multi-Agent 协作架构设计**

设计了 CrawlerAgent / EditorAgent / QAAgent / PublisherAgent 4 个专业 Agent，通过 MultiAgentOrchestrator 编排器协调。Agent 间通过统一的 AgentMessage 协议通信，每个 Agent 有独立角色定义、工具集、状态报告。基于 LangGraph 将流程建模为有状态图（recommend→search_origin→generate→qa_check→publish），支持条件路由与错误恢复。每步记录了 reasoning（决策原因）和耗时到 agent_steps 审计表。

**2. RAG 语义检索 + 知识图谱引擎**

基于 BAAI/bge-small-zh-v1.5（512 维）对 737 条公告原文做 Embedding，FAISS 向量索引。实现混合检索：向量语义 + 关键词精确 + RRF 双路融合，支持"不限专业""快截止的教师岗"等口语化查询。在 SQLite 上构建轻量图数据库，自动抽取 1524 节点、2211 条关系，支持同地区/同类型公告的关联推荐。

**3. Agent 记忆系统与自我进化**

实现三层记忆架构：短期记忆（会话上下文）、长期记忆（跨会话经验）、学习模式（从失败步骤中自动提取经验教训）。支持记忆衰减、关键词+向量混合检索。预置 6 条种子最佳实践，Agent 执行后可自动学习。

**4. 人机协同审批 + 成本追踪 + 诊断恢复**

构建正式审批状态机（DRAFT → PENDING_APPROVAL → APPROVED → PUBLISHED），支持驳回重新生成、超时自动处理。Token 成本可按模型/任务/天多维度追踪。自建诊断引擎覆盖 12 种微信 API 错误码 + 8 类通用异常模式，每种错误匹配可能原因、修复建议、自动恢复策略。

**5. 大模型内容生成 + 质量保障**

基于公告结构化字段自动生成公众号文章，内置标题模板引擎和"适合人群提醒"模块（覆盖 8 种考试类型）。接入 DeepSeek 进行草稿质检，配套事实一致性评测脚本，从招聘人数、地区、截止日期、原文链接、无关内容 5 个维度量化评测。质检不通过自动阻断发布。

**6. Web 智能对话面板**

实现 ChatGPT 风格对话界面，SSE 流式推送实时显示 AI 思考过程和工具调用状态。RAG + KG 增强回复，多轮对话上下文管理，对话历史本地持久化。支持自然语言查询、公告推荐、Agent 任务触发等多种交互模式。

**7. 多源数据采集与附件智能解析**

设计粉笔 API 和公考雷达（Playwright）双源采集，支持 18 种考试类型，Hash 去重，截止日期自动修正。附件智能解析（.xlsx/.docx/.pdf），岗位表自动筛选并渲染为图片。原文覆盖率 96.5%。
