# 考公信息运营 Agent

**个人项目** ｜ 2025.06 – 至今

**技术栈：** Python、SQLite、FAISS、DeepSeek API、LangGraph、BGE Embedding、NetworkX、Playwright、SSE、HTML/CSS/JS、微信公众号 API

**项目地址：** https://github.com/w11aaa/zdhwxgzh

---

## 项目概述

面向微信公众号内容运营场景的智能 Agent 系统。自动从粉笔、公考雷达等多源采集公务员/事业单位/教师/国企等招聘公告（已入库 737 条），基于 RAG 语义搜索 + 知识图谱构建智能对话系统，经 Multi-Agent 协作完成内容生成全流程，支持 Web/微信双端对话。

---

## 核心职责与技术实现

**1. RAG 语义搜索引擎**

基于 BAAI/bge-small-zh-v1.5（512 维）对 737 条公告原文做 Embedding，FAISS Inner Product 向量索引。实现混合检索：向量语义 + 关键词精确 + RRF（Reciprocal Rank Fusion）双路融合。用户可用自然语言搜索"海淀区有没有计算机类的事业编"，语义理解"体制内""不限专业"等口语表达。搜索结果自带来源引用。

**2. 知识图谱引擎**

在 SQLite 上构建轻量图数据库，从 737 条公告中自动抽取实体节点（地区/单位/公告/考试类型/岗位）和关系边（BELONGS_TO/PUBLISHES/HAS_CATEGORY/SIMILAR_TO），共 1524 节点、2211 条关系。支持关联推荐（同地区同类型公告发现）、实体搜索、单位画像。

**3. Multi-Agent 协作架构**

设计 CrawlerAgent（采集+搜索）/ EditorAgent（内容生成）/ QAAgent（质检+诊断）/ PublisherAgent（发布）4 个专业 Agent，通过 MultiAgentOrchestrator 编排器协调。Agent 间通过统一消息协议（AgentMessage）通信，每个 Agent 有独立角色定义、工具集、状态报告。

**4. LangGraph 状态机工作流**

将 Agent 流程建模为有状态图：recommend → search_origin → generate → qa_check → publish，支持条件路由（质检通过→发布 / 失败→人工审批）和错误恢复。每步记录节点结果和耗时。

**5. Agent 记忆与学习系统**

三层记忆架构：短期记忆（会话上下文）+ 长期记忆（跨会话经验）+ 学习模式（运行中提取）。支持记忆衰减、关键词+向量混合检索、自动从失败步骤中提取经验教训。预置 6 条种子最佳实践。

**6. 人机协同审批状态机**

DRAFT → PENDING_APPROVAL → APPROVED → PUBLISHED，支持驳回→重新生成（REWORK），超时自动处理，审批原因记录。正式的状态机设计，完整的状态转换规则。

**7. 大模型内容生成与质量保障**

基于公告结构化字段 + 原公告 HTML 自动生成公众号文章，内置标题模板引擎和"适合人群提醒"模块（覆盖 8 种考试类型）。接入 DeepSeek 进行草稿质检，配套事实一致性评测脚本（5 维度 100% 通过率）。Token 成本追踪（按模型/任务/天分组）。

**8. Web 智能对话面板**

ChatGPT 风格深色主题对话界面，SSE 流式推送实时显示 AI 打字过程。RAG+KG 增强回复，Tool Badge 展示检索状态。多轮对话上下文管理（session-based），对话历史本地持久化。

**9. 失败自动诊断与恢复**

自建诊断引擎覆盖 12 种微信公众号 API 错误码 + 8 类通用异常模式，每种错误匹配可能原因、修复建议、自动恢复策略。6/6 测试用例全部命中。

**10. 多源数据采集与附件处理**

双源采集（粉笔 API + 公考雷达 Playwright），Hash 去重，截止日期自动修正。附件智能解析（.xlsx/.docx/.pdf），岗位表自动筛选并渲染为图片。原文覆盖率 96.5%。

## 项目亮点（面试建议话术）

> 1. **完整的 Agent 工程体系**：多 Agent 协作 + 记忆系统 + 审批状态机 + 成本追踪 + 诊断恢复，不是简单的"调 API 聊天机器人"。
>
> 2. **RAG + 知识图谱双引擎**：不是简单的关键词搜索。向量语义检索理解"体制内""不限专业"等口语表达，知识图谱关联推荐跨公告发现。
>
> 3. **可审计的 Agent 决策**：每一步工具调用记录了 reasoning（为什么调这个工具）、参数、结果和耗时，面试时可以直接展示 agent_steps 表。
>
> 4. **工程可靠性设计**：评测脚本量化内容质量，诊断模块让系统从"报错"升级为"能解释为什么出错并给出修复建议"，Token 追踪控制成本。
>
> 5. **从 LangGraph 到自研编排器**：理解状态机工作流原理，能同时使用框架和自研方案，不是只会调包。
